Summary: Latitude vs. X

By following the analytics paradigm, the “Latitude Weather Analysis” project was completed to visualize the weather of 771 cities around the world of varying distance from the equator.

A data analysis was conducted to answer the following question:

“What’s the weather like as we approach the equator?”

Latitude and longitude coordinate pairs were randomly generated to identify the location of a city closest to each coordinate pair by using the Citipy open source library. For each randomly selected city, weather data was then collected through the OpenWeatherMap API for four metrics: maximum temperature, humidity, cloudiness, and wind speed.

Thereafter, on a jupyter notebook Python Pandas and Numpy library was used to assemble and clean data. Also, Matplotlib and Seaborn libraries were used to generate graphs from the created data frames for (1) Latitude vs. Maximum Temperature, (2) Latitude vs. Humidity, (3) Latitude vs. Cloudiness, (4) Latitude, vs. Wind Speed. Observable trends were identified for each plot. In depth explanation can be read by clicking one of the metrics in the Plot dropdown menu on the navigation bar or by clicking on one of the graphs under the Visualization section.

Limitations to the data analysis included but not limited to an api call conducted only on 6/23/2018, Citipy open source library does not contain all cities/towns in the world, and no statistical testing conducted to conclude correlations between collected metrics.

Based on the “Latitude Weather Analysis”, observable trends can be deduced from the each of the plots. The maximum temperature increases as the city’s location is closer to the equator. Regardless of the distance from the equator the concentration of the number of cities decreases as the measured humidity (%) decreases from 100% to 0%, the distribution of measured cloudiness (%) for cities is well-dispersed, and the concentration of the number of cities increases as the measured wind speed (mph) increases from 0 mph to 35 mph.
